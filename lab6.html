Lab 6: Working with Redshift
Connection, data download and upload

Step 1

Setup

In this next lab, we’ll create a redshift cluster, and use the notebook instructions to create a new redshift cluster, add csv data to it, and query it.

We’ll be using the "iris.csv" dataset from the UCI machine learning repository that can be found in here - https://archive.ics.uci.edu/ml/datasets/iris.

TWe will not be training a model here, but simply doing the preprocessing (transferring "training data" from redshift to S3, and the post-processing (uploading inference data back to S3) .

Clone the following repository in your already created notebook instance:
git clone https://github.com/w601sxs/Sagemaker-Redshift-Example-Notebook.git

Step 2

Browse to the folder you just cloned and follow instrunctions in the notebook

Notes on broad steps:
  • Create VPC
  • Create Redshift cluster subnet group ahead of time in same VPC with inbound access on 5439 and make sure to include vpc subnet, along with no public access and enhanced vpc
  • While creating a new Redshift cluster try to avoid dc2, prefer ds.2xl instead
  • Create Sagemaker instance in same vpc, vpc subnet and redshift subnet group as redshift cluster
  • If referencing GitHub repo directly, would need to be in public subnet with internet access
